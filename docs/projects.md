# Student Projects

The Neural Cognitive Mapping (NCM) Lab at the University of Ottawa offers student research projects related to the 3C Platform. Projects are available for undergraduates through **CSI 4900**, **PSY 4276**, or **HSS 4901** course credit.

If you are interested in contributing, contact [Jason Steffener](mailto:jsteffen@uottawa.ca).

---

## High priority

### Automatic scoring of paper-based questionnaires

Many validated clinical instruments were originally designed for paper administration. This project implements [SDAPS](https://sdaps.org/) — an open-source optical mark recognition tool — into the 3C Platform so that completed paper forms can be scanned and scored automatically alongside digitally administered assessments.

**Skills useful:** Python, image processing, Linux shell

---

### Integration of 3C Platform results into REDCap

[REDCap](https://www.project-redcap.org/) is the most widely used research data management system in clinical research. This project builds a bridge between JATOS result exports and REDCap so that 3C Platform scores flow directly into a researcher's REDCap project without manual data entry.

**Skills useful:** JavaScript or Python, REST APIs, JSON

---

## Lower priority

### Additional continuous performance cognitive tasks

Continuous performance tasks (CPTs) measure sustained attention over several minutes of repetitive stimuli. The platform currently includes discrete trials; this project identifies validated CPT paradigms from the literature and implements them within the jsPsych framework used by the 3C Platform.

**Skills useful:** JavaScript, jsPsych, cognitive neuroscience background helpful

---

### Cognitive screening based on speech and language analysis

Subtle changes in speech fluency, word-finding, and sentence structure are increasingly recognised as early markers of cognitive decline. This project surveys the current literature and implements speech or language analysis tools — potentially using large language models — that can be integrated as a scoring layer on top of the platform's existing speech-recognition response type.

**Skills useful:** Python, NLP / LLMs, signal processing or linguistics background helpful

---

### When is enough, enough? — Adaptive stopping criteria

Adaptive tasks (digit span, delayed match-to-sample) already adjust difficulty trial-by-trial. This project extends that logic to the battery level: defining criteria by which the platform can determine that sufficient data has been collected and offer an early-exit path, reducing burden on participants who are clearly above or below a threshold.

**Skills useful:** JavaScript, psychometrics or statistics background helpful
